ISRO Launch Success Prediction

Data-driven launch risk assessment for India's space program — helping commercial operators make informed mission decisions

🎯 Problem / Motivation

The Challenge: Commercial satellite operators face enormous costs (crores per mission) with limited ways to assess risk beyond broad historical success rates.

Why It Matters: Recent anomalies — PSLV-C61 (May 2025) and PSLV-C62 (Jan 2026) — showed that traditional "90% success rate" metrics fail to capture mission-specific risks based on payload weight, orbit type, and vehicle configuration.

Business Impact: Insurance companies and commercial operators need granular, feature-driven probability scores to make data-backed decisions about mission budgeting and risk management.

🛠️ Approach

Data Collection: Scraped Wikipedia launch tables for PSLV, GSLV, LVM3 missions. Built custom parser to handle multi-satellite missions and complex HTML rowspan structures.

Data Cleaning: Forward-fill technique for sub-satellite rows, regex-based payload mass extraction and summing, filtering to completed missions only (Success/Failure/Partial).

Feature Engineering: Created categorical features: Vehicle Family, Orbit Type, Launch Site. Numerical: Total Payload Mass (kg). Mapped Partial Failures → Failure for strict binary classification.

Class Balancing: Applied SMOTE (Synthetic Minority Over-sampling) to address 8:1 success-to-failure imbalance and prevent model bias toward "always predict success."

Model Training: Random Forest Classifier with GridSearchCV (5-fold CV) for hyperparameter tuning. Optimized for recall on failure class to minimize false negatives in risk assessment.

Validation: Included most recent data through Jan 2026 (PSLV-C62) to ensure model reflects current hardware state.

💡 Key Insights / Results

1. Feature Importance: Vehicle Family and Payload Mass are dominant predictors. GTO missions with heavy payloads show distinct risk profiles vs. LEO missions.

2. Model Performance: Random Forest achieved 85%+ test accuracy with balanced recall/precision. SMOTE successfully prevented "always success" predictions.

3. Real-World Validation: Model correctly flagged higher risk for PSLV-C62 mission profile (Jan 2026 anomaly), demonstrating practical applicability.

4. Actionable Output: Instead of binary predictions, model outputs probability scores (e.g., "78% success chance") for nuanced risk assessment.

🔮 Reflection / Next Steps

What I Learned: Scraped messy real-world data, handled severe class imbalance, understood domain constraints matter for ML interpretability.

🚀 Roadmap to v2.0: Physics-Aware Constraints

Current Limitation: v1.0 predicts success probability for ANY input — even physically impossible missions (e.g., 10-ton payload on PSLV to GTO).

Planned Enhancement: Integrate rocket-specific payload capacity curves:

  • PSLV payload limits by orbit (LEO: 3.8T, SSO: 1.75T, GTO: 1.4T)
  • GSLV Mk II/Mk III performance envelopes
  • LVM3 configuration-specific constraints

Technical Approach: Pre-validation layer checks mission feasibility against rocket specs before invoking ML model. Returns "Mission Not Feasible" if parameters exceed vehicle capability.

Impact: Moves model from "probability predictor" to "mission advisor" — realistic for commercial operators planning actual launches.

Future Extensions: Add weather data (wind speed, temperature), compare ISRO vs. SpaceX reliability, deploy live Streamlit app with v2.0 constraints.
